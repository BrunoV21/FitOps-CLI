    `GET /race/{course_id}` — course profile:
    - course = await get_course(course_id)
    - If None: 404 HTMLResponse
    - segments = course.get_km_segments()
    - Build elevation_data for Chart.js: labels=[seg["km"] for seg in segments], elevations=[...computed from points]
    - Actually: elevation profile needs course points, not just segments. Load course.get_course_points() → compute elevation at each km start.
    - elevation_labels = [str(s["km"]) for s in segments]
    - elevation_data = [s.get("elevation_m_start", 0.0) for s in segments]  — NOTE: segments from build_km_segments don't have elevation_m_start. Instead, compute from course_points: group by km bucket, take first point's elevation_m.
    - Simpler approach: use course_points directly, sample at distance intervals.
    - Best approach (match research doc): pass the segments list to template; template builds elevation as cumulative elevation_delta within each km bucket. Template shows km on X, elevation gain per km as a bar, or running elevation as a line.
    - For a running elevation line: pass elevation_at_km = [p["elevation_m"] for p in course_points if p matches km boundary] — compute in route.
    - Implementation: iterate course_points, record elevation_m at each whole-km boundary:
      ```python
      points = course.get_course_points()
      elevation_profile = []  # [{km, elevation_m}]
      for km_idx in range(len(segments) + 1):
          target_dist = km_idx * 1000.0
          # Find closest point to this distance
          closest = min(points, key=lambda p: abs(p["distance_from_start_m"] - target_dist))
          elevation_profile.append({"km": km_idx, "elevation_m": round(closest["elevation_m"], 1)})
      ```
    - template context: course summary dict, segments, elevation_profile as JSON for Chart.js

    `GET /race/{course_id}/simulate` — simulation form (GET shows blank form, POST runs simulation):
    - GET: load course, render form template with course context
    - POST (Form data): target_time, strategy, pacer_pace, drop_at_km, temp, humidity, wind, wind_dir
    - Parse inputs, call simulate_splits or simulate_pacer_mode
    - On ValueError: render form again with error message
    - On success: render simulate.html with splits, pacer data, chart data

    **POST form handling:** Use `from fastapi import Form` for form fields. All form fields are Optional[str] with None defaults.

    ```javascript
    callbacks: {
      label: function(context) {
        const s = context.raw;
        const m = Math.floor(s / 60);
        const sec = Math.round(s % 60);
        return `${m}:${sec.toString().padStart(2,'0')}/km`;
      }
    }
    ```
